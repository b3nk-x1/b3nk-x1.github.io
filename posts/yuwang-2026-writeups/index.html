Crypto

ECDSA Nonce Reuse

ECDSA Nonce Reuse 私钥恢复题

1.获取到 challenge.json 文件

解压 otp_09.zip 后得到 challenge.json。文件中给出 public_key_x、public_key_y、message1、message2、signature1、signature2 和 curve。这里 curve 为 SECP256k1。

重点看两组签名参数，signature1_r 和 signature2_r 完全相同，但 s 值不同，说明两条不同消息复用了同一个 ECDSA nonce k。

2.然后利用 CyberChef 查看消息内容

把 message1 按 Hex 解码，得到明文 Welcome to the CTF challenge!。

图2 CyberChef 将 message1 从 Hex 解出明文。

把 message2 按 Hex 解码，得到明文 Please recover the secret flag.。两条消息不同，但签名 r 相同，可以按 nonce reuse 继续做。

图3 CyberChef 将 message2 从 Hex 解出明文。

3.再利用脚本恢复 nonce 和私钥

ECDSA 签名公式为 s = k^(-1) * (z + r*d) mod n。两条消息复用同一个 k 时，可以先消去私钥项恢复 k，再代回恢复私钥 d。

PyCharm 运行 solve.py，脚本先检查 r1 == r2，再计算 z1、z2、k、d，并用 d * G 反推公钥。

4.然后解出 flag

脚本恢复出的私钥 d 为：

a46b8f59aac5f0d5f1e661827c1cf3a7f536c9ce5a5d2452942215f16480d48b

计算出的公钥和 challenge.json 中的 public_key_x / public_key_y 一致，public key ok 为 True，说明私钥恢复正确。题目 flag 取私钥 hex 的前 32 位。

最终 flag：

flag{ecdsa_nonce_reuse_a46b8f59aac5f0d5f1e661827c1cf3a7}

ScatterRSA

1. 获取题目文件

附件为 ScatterRSA9.zip，解压后得到 task.py 和 output.txt。task.py 是加密逻辑，output.txt 是三组公开参数和密文。

核心加密代码如下：

m = bytes_to_long(flag)

e = 3

for i in range(3):

p = getPrime(512)

q = getPrime(512)

n = p * q

a = random.getrandbits(128) | (1 << 127)

b = random.getrandbits(256) | (1 << 255)

c = pow(a * m + b, e, n)

图1 task.py 中明文先做 a*m+b，再用 e=3 加密。

output.txt 中给出了 e=3 以及三组 n、a、b、c 参数。

图2 output.txt 中的三组公开参数和密文。

可以看出同一份明文 m 被放进三条不同信道中，每次都有不同的 n、a、b，并且都使用低指数 e=3。

2. 分析 RSA 结构

每组数据满足：

c_i = (a_i * m + b_i)^3 mod n_i

因此可以构造三条关于同一个未知数 x 的同余：

f_i(x) = (a_i * x + b_i)^3 - c_i ≡ 0 mod n_i

三组模数 n_i 不同，但明文 x 相同。把三条同余用 CRT 合并后，可得到模 N=n1n2n3 下的三次多项式 F(x)。

flag 长度较短，转成整数后小于给定界，使用 Coppersmith 小根攻击求出 x，也就是原始明文 m。

3. 编写 EXP

将下面脚本保存为 1.py，并和 output.txt 放在同一目录下运行。脚本自动读取参数，不需要手动粘贴大整数，也没有写死 flag。

import re, math

from pathlib import Path

from fractions import Fraction

from datetime import datetime

from sympy import symbols, Poly, ZZ

from sympy.polys.polytools import ground_roots

x = symbols(‘x’)

def long_to_bytes(n):

return int(n).to_bytes((int(n).bit_length() + 7) // 8, ‘big’)

def crt(rs, ms):

M = math.prod(ms)

s = 0

for r, m in zip(rs, ms):

Mi = M // m

s = (s + r * Mi * pow(Mi, -1, m)) % M

return s

def eval_mod(co, v, mod):

s = 0

for c in co[::-1]:

s = (s * v + c) % mod

return s

def gs(B):

n, m = len(B), len(B[0])

mu = [[Fraction(0) for _ in range(n)] for __ in range(n)]

G = [[Fraction(0) for _ in range(m)] for __ in range(n)]

norm = [Fraction(0) for _ in range(n)]

for i in range(n):

G[i] = list(map(Fraction, B[i]))

for j in range(i):

if norm[j]:

mu[i][j] = sum(B[i][k] * G[j][k] for k in range(m)) / norm[j]

G[i] = [G[i][k] - mu[i][j] * G[j][k] for k in range(m)]

norm[i] = sum(t * t for t in G[i])

return mu, norm

def nearest(q):

n = q.numerator // q.denominator

r = q - n

if r > Fraction(1, 2): n += 1

if r < Fraction(-1, 2): n -= 1

return n

def lll(B, delta=Fraction(3, 4)):

B = [list(map(int, r)) for r in B]

mu, norm = gs(B)

k = 1

while k < len(B):

for j in range(k - 1, -1, -1):

q = nearest(mu[k][j])

if q:

B[k] = [B[k][i] - q * B[j][i] for i in range(len(B[0]))]

mu, norm = gs(B)

if norm[k] >= (delta - mu[k][k-1] ** 2) * norm[k-1]:

k += 1

else:

B[k], B[k-1] = B[k-1], B[k]

mu, norm = gs(B)

k = max(k - 1, 1)

return B

def main():

data = Path(‘output.txt’).read_text(encoding=’utf-8’)

v = {k: int(n) for k, n in re.findall(r’([enabc]\d?)\s*=\s*(\d+)’, data)}

e = v[‘e’]

ns = [v[f’n{i}’] for i in range(1, 4)]

aa = [v[f’a{i}’] for i in range(1, 4)]

bb = [v[f’b{i}’] for i in range(1, 4)]

cc = [v[f’c{i}’] for i in range(1, 4)]

N = math.prod(ns)

print(‘========== 1. 读取参数 ==========’)

print(‘[+] e =’, e)

print(‘[+] n bits =’, [n.bit_length() for n in ns])

print(‘[+] a bits =’, [a.bit_length() for a in aa])

print(‘[+] b bits =’, [b.bit_length() for b in bb])

coeff = []

for k in range(4):

rs = []

for a, b, c, n in zip(aa, bb, cc, ns):

if k == 0:

t = pow(b, 3, n) - c

elif k == 1:

t = 3 * a * pow(b, 2, n)

elif k == 2:

t = 3 * pow(a, 2, n) * b

else:

t = pow(a, 3, n)

rs.append(t % n)

coeff.append(crt(rs, ns))

inv = pow(coeff[3], -1, N)

coeff = [(i * inv) % N for i in coeff]

cint = [i - N if i > N // 2 else i for i in coeff]

cint[3] = 1

print(‘\n========== 2. 构造 CRT 多项式 ==========’)

print(‘[+] N bits =’, N.bit_length())

print(‘[+] degree(F) = 3, leading coefficient = 1’)

print(‘\n========== 3. Coppersmith 小根 ==========’)

X = 2 ** 320

print(‘[+] root bound X = 2^320’)

f = Poly(sum(cint[i] * x ** i for i in range(4)), x, domain=ZZ)

polys = [Poly(N * (x * X) ** i, x, domain=ZZ) for i in range(3)]

polys.append(Poly(f.as_expr().subs(x, x * X), x, domain=ZZ))

L = [[int(p.coeff_monomial(x ** i)) for i in range(4)] for p in polys]

print(‘[+] Lattice: 4 x 4’)

roots = []

for idx, row in enumerate(lll(L)):

h = Poly(sum((row[i] // (X ** i)) * x ** i for i in range(4)), x, domain=ZZ)

for r in ground_roots(h):

r = int(r)

if 0 <= r < X and eval_mod(coeff, r, N) == 0 and r not in roots:

print(f’[+] Small root found from LLL row {idx}’)

roots.append(r)

print(‘\n========== 4. 验证并输出 flag ==========’)

for m in roots:

flag = long_to_bytes(m)

print(‘[+] m bits =’, m.bit_length())

print(‘[+] F(m) mod N =’, eval_mod(coeff, m, N))

print(‘[+] bytes =’, repr(flag))

print(‘[+] flag =’, flag.decode())

print(‘[+] time =’, datetime.now().strftime(‘%Y-%m-%d %H:%M:%S’))

if name == ‘main‘:

main()

4.然后解出 flag

ChaCha20 属于流密码，加密过程本质上是明文和密钥流异或。已知目标密文、key、nonce、counter 后，生成同样的 ChaCha20 密钥流，再与密文异或即可还原 flag。

最终使用 Python 复现 ChaCha20 block，并对目标密文进行异或还原。脚本里再把结果复加密一次，确认与 target_hex 一致。EXP 如下：

import struct

TARGET_HEX = “d097c3f6d27e8c28ff2fa736bae486998e118c7624f0b890c41351cec30c2c1cd675a39ffbd9”

KEY_HEX = “149263a16f2d89cbf0375b1ca94e78d3226017ee9abc4d0853e1762a8dc4903f”

NONCE_HEX = “44332211abcdef668899aa55”

COUNTER = 1

def rotl32(x, n):

return ((x << n) & 0xffffffff) | (x >> (32 - n))

def quarter_round(s, a, b, c, d):

s[a] = (s[a] + s[b]) & 0xffffffff

s[d] ^= s[a]

s[d] = rotl32(s[d], 16)

s[c] = (s[c] + s[d]) & 0xffffffff

s[b] ^= s[c]

s[b] = rotl32(s[b], 12)

s[a] = (s[a] + s[b]) & 0xffffffff

s[d] ^= s[a]

s[d] = rotl32(s[d], 8)

s[c] = (s[c] + s[d]) & 0xffffffff

s[b] ^= s[c]

s[b] = rotl32(s[b], 7)

def chacha20_block(key, counter, nonce):

const = b”expand 32-byte k”

state = list(

struct.unpack(“<4I”, const)

  • struct.unpack(“<8I”, key)

  • (counter,)

  • struct.unpack(“<3I”, nonce)

)

working = state[:]

for _ in range(10):

quarter_round(working, 0, 4, 8, 12)

quarter_round(working, 1, 5, 9, 13)

quarter_round(working, 2, 6, 10, 14)

quarter_round(working, 3, 7, 11, 15)

quarter_round(working, 0, 5, 10, 15)

quarter_round(working, 1, 6, 11, 12)

quarter_round(working, 2, 7, 8, 13)

quarter_round(working, 3, 4, 9, 14)

result = [(working[i] + state[i]) & 0xffffffff for i in range(16)]

return struct.pack(“<16I”, *result)

def chacha20_xor(data, key, nonce, counter=1):

out = bytearray()

for pos in range(0, len(data), 64):

stream = chacha20_block(key, counter, nonce)

chunk = data[pos:pos + 64]

out.extend(a ^ b for a, b in zip(chunk, stream))

counter += 1

return bytes(out)

def main():

target = bytes.fromhex(TARGET_HEX)

key = bytes.fromhex(KEY_HEX)

nonce = bytes.fromhex(NONCE_HEX)

print(“[1] 参数检查”)

print(f”target_hex = {TARGET_HEX}”)

print(f”target_len = {len(target)} bytes”)

print(f”key = {KEY_HEX} ({len(key)} bytes)”)

print(f”nonce = {NONCE_HEX} ({len(nonce)} bytes)”)

print(f”counter = {COUNTER}”)

print(“\n[2] ChaCha20 解密”)

flag_bytes = chacha20_xor(target, key, nonce, COUNTER)

flag = flag_bytes.decode()

print(f”flag_bytes = {flag_bytes!r}”)

print(f”flag = {flag}”)

print(“\n[3] 复加密验证”)

verify_hex = chacha20_xor(flag_bytes, key, nonce, COUNTER).hex()

print(f”verify_hex = {verify_hex}”)

print(f”match = {verify_hex == TARGET_HEX}”)

print(“\n[4] 最终 flag”)

print(flag)

if name == “main“:

main()

3.构造 Fastbin 重叠

选择 len=0x18，是因为密码 chunk 和 user 结构体 chunk 在 glibc 2.23 下都会进入同一个 0x20 fastbin。

Delete 后 fastbin 链大致为 A.user -> A.data。重新 Register(1, 1) 时，B.data 会拿到旧的 A.user 位置。

这样 users[0] 仍然指向旧 A.user，而 B.data 也占着这块内存，所以通过 Edit(1) 可以改 A.user 的 data / p / size。
