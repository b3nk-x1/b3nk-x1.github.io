图 2 附件文件和 Windows 可复现工具

利用 Wireshark 查看 DNS 可疑记录

在 Wireshark 中使用过滤器 dns 或 dns.qry.name contains “exfil-cdn.com”,可以看到可疑域名 secret.exfil-cdn.com 的 TXT 记录。TXT 内容为 TW9yZVNlY3VyZUFlczEyOA==,使用 Base64 解码后得到 MoreSecureAes128,长度正好是 16 字节,可作为 AES-128 密钥。

图 3 DNS TXT 记录提取和 AES key

利用 Wireshark 和 Python 分析 ICMP 隐蔽信道

在 Wireshark 中使用过滤器 ip.src == 10.10.20.33 && ip.dst == 45.76.188.23 && icmp.type == 8,可以看到 10.10.20.33 向 45.76.188.23 发送了大量 ICMP Echo Request。继续检查字段发现这些包的 TTL 末位只取 0 和 1,因此按 ICMP seq 排序后取 TTL & 1,每 8 bit 还原 1 字节。

图 4 ICMP TTL bit 还原出的密文和算法提示

利用编码和 AES 解出 flag

TTL 隐写还原出的内容包含一段 hex 密文和提示 AES-ECB, key is in DNS TXT。因此将 hex 转 bytes,使用 DNS TXT 中得到的 MoreSecureAes128 作为 AES-ECB 密钥解密,即可得到最终 flag。

seq=END 中给出了校验信息:archive_size=10601;sha256=4019ef293eeb8af8a0d5c0e66c0bdb3cbcd17067839f42c86beb501a83d6c59b。将 00001-00070 的 note 字段按序拼接后 Base64 解码,得到 10601 字节文件,文件头为 PK,说明是 ZIP 文件,并且 SHA256 与 END 中一致。

LevelLedger

项目 内容
题目名称 LevelLedger
WP范围 只记录从附件生成 classified_data.csv 的过程,不包含上传平台、验证码识别和 flag 截图。
复现环境 Windows 10/11、7-Zip、Python 3.10+、PowerShell 或 CMD、任意文本编辑器。
使用工具 7-Zip 解压;Python csv/base64/re/datetime/unicodedata 标准库;cryptography 库用于 RSA 私钥解密。

附件解压后可以看到 mixed_data.csv、leaked_private.pem、分类分级规范.md 三个关键文件。mixed_data.csv 只有 data_id 和 value 两列,value 中混有明文、分隔符样式数据、全角字符以及 RSA 密文。

按照规范,最后需要输出 data_id,value,category,level 四列,其中 value 必须是解密和归一化后的明文。

工具准备

  • 使用 7-Zip 或 Windows 自带解压功能展开附件。
  • 使用 Python 3 处理 CSV、Base64、日期校验和 Luhn 校验。
  • 安装 cryptography 库用于读取 leaked_private.pem 并执行 RSA OAEP 解密。

字段处理规则

    1. 先判断 value 是否为 344 字符 Base64 串;若是,使用 leaked_private.pem 进行 RSA OAEP-SHA256 解密。
    1. 对解密后的字符串做 NFKC 归一化,把全角数字转成半角数字。
    1. 去掉空白和横线分隔符,例如银行卡、身份证、手机号、UUID、CARD-xxxx 等都统一得到无空格/横线的明文。
    1. 手机号若带 +86 或 0086 前缀,去掉国家码后再判断 11 位手机号。
    1. 按 idcard、bankcard、phone、ip、normal 的顺序分类,并写入对应 S4/S4/S3/S2/S1 等级。

关键判定

  • 身份证:18 位,出生日期合法,最后一位满足 GB 11643 加权校验码。
  • 银行卡:16 到 19 位纯数字,Luhn 校验和能被 10 整除。
  • 手机号:11 位纯数字,前三位在规范给出的大陆手机号段集合内。
  • IPv4:四段点分十进制,0 到 255,无前导零,无 CIDR 后缀。
  • 其他不满足规则的数据统一归为 normal / S1。

MaskTrace

题目附件中给出 user_export.sql 和《脱敏规范.md》。目标是只筛选 status=active 的用户,按 id 升序导出固定表头的 UTF-8 CSV。整个过程可在普通 Windows 环境完成,使用工具为 7-Zip、Python 3、文本编辑器/Excel,脚本只调用 Python 标准库。

1. 解压并查看附件

  • 使用 7-Zip 解压“7:MaskTrace的附件.zip”
  • 继续解压其中的“MaskTrace附件.zip”,得到“脱敏规范.md”和“user_export.sql”
  • 用记事本、VS Code 或 Notepad++ 打开规范文件,确认输出表头和逐字段校验规则
  • 查看 SQL 文件可以发现其为 SQLite 风格的建表语句与 INSERT 数据

2. 导入 SQL 并筛选 active 数据

可以用 DB Browser for SQLite 执行 SQL,也可以直接使用 Python 标准库 sqlite3 在内存数据库中执行。筛选时注意 id 在 SQL 中以文本形式出现,为保证 1、2、10 等顺序正确,需要按整数排序。

3. 按规范逐字段校验与脱敏

字段 校验重点 脱敏输出
username 按长度处理,保持原大小写 首尾保留,中间替换为 *;长度 2 保留首字符
password 不做格式校验 原文 MD5,输出 32 位小写 hex
name 必须全中文且长度不少于 2 二字保留姓;三字及以上保留首尾;非法为 INVALID
idcard 18 位、出生日期真实、末位校验码正确 只保留出生年份,其余为 *;非法为 INVALID
phone 去掉空格、短横线、+86 等格式差异后校验号段 152****6242 格式;非法为 INVALID
email 先统一转小写,再判断邮箱格式 用户名按 username 规则脱敏,域名保留
bankcard 去掉空格/短横线后必须是 16-19 位数字 保留前 6 后 4,中间替换为 *
address 必须包含中文“号” 保留第一个“号”之前内容,后续统一为 ***
ip 必须为合法 IPv4 最后一段替换为 *
birthday 支持 YYYY-MM-DD 和 YYYY/MM/DD,日期必须真实 以 2026-05-01 计算周岁,输出十年年龄段

1. 工具环境

  • Windows 10/11:使用资源管理器、PowerShell 或 7-Zip 解压附件。

3. 按 README 规则判断攻击类型

攻击类型只根据 user_input 判断,并按题目给出的优先级处理:encoding > unicode > role_play > direct。

类型 判断依据
encoding 输入中出现 Base64、Decode、编码指令等提示,并带有一段较长 Base64 字符串。
unicode 输入中出现 \uXXXX 形式的 Unicode 转义序列。
role_play 输入中出现“假设你是、你现在是、扮演、Act as、You are now、Let’s play”等角色设定。
direct 输入中直接出现忽略安全规则、系统管理员、调试命令、dump all、print PII 等覆盖安全策略的指令。

4. 提取并校验模型响应中的 PII

只处理攻击轮次中的 model_output。普通客服对话即使出现订单号、会员号、客服邮箱,也不写入结果。

PII 类型 提取和校验方式
idcard 匹配 18 位身份证号,校验出生日期和最后一位校验码;字符分隔形式先去掉空格、连字符或逗号。
phone 匹配 11 位数字,并检查前三位是否在 README 给出的手机号段集合中。
bankcard 匹配 16-19 位数字,先检查常见 BIN 前缀,再使用 Luhn 算法校验。
email 使用标准邮箱正则匹配,Base64 输出先解码后再匹配。

模型响应中的数据可能有三种变形:明文、Base64 编码、字符分隔。脚本处理时分别进行直接匹配、Base64 解码、去分隔符还原。
